# 记录正文标记语法

记录正文统一使用 `[[type:参数1|参数2]]`。新内容不要再使用旧式花括号、圆括号、感叹号、箭头或单字符上下标；渲染器暂时保留旧语法兼容。

| 功能 | 推荐写法 |
|---|---|
| 人物 | `[[person:人物ID|显示文字]]` |
| 额外记录人 | `[[author:人物ID|显示文字]]` |
| 术语 | `[[term:术语ID|显示文字]]` |
| 记录跳转 | `[[record:文件名|显示文字]]` |
| 分式 | `[[frac:上方文字|下方文字]]` |
| 注解 | `[[anno:注解内容|被注释文字]]` |
| 插图 | `[[illu:example.png|显示文字]]` |
| 删除线 | `[[del:文字]]` |
| 下划线 | `[[under:文字]]` |
| 标红 | `[[red:文字]]` |
| 黑幕 | `[[hide:文字]]` |
| 上标 | `[[sup:文字]]` |
| 下标 | `[[sub:文字]]` |
| 箭头上下结构 | `[[arrow:上方文字|下方文字]]` |
| 居中 | `[[center:文字]]` |
| 右对齐 | `[[right:文字]]` |

参数内容可以嵌套其他统一标记。例如：

```text
[[anno:注解中包含 [[person:zhx|小张]]|被注释文字]]
[[frac:[[del:上方文字]]|[[under:下方文字]]]]
```

`[[person:...]]`（以及兼容的 `[[人物ID|显示文字]]`）表示事件参与者；`[[author:...]]` 表示正文中补充的额外记录人。额外记录人与记录 JSON 的 `author` 字段共同计入记录事件，但不会仅因 `author` 标记而计入参与事件。

## 转义

标记参数中的特殊字符使用反斜杠转义：

```text
\|  普通竖线
\[  普通左方括号
\]  普通右方括号
\\  普通反斜杠
```

JSON 字符串还需要转义反斜杠。例如正文中的 `A|B` 可写为：

```json
{
    "content": "[[anno:A\\|B|标签]]"
}
```

## 参数限制

- 人物和术语 ID 只能包含英文字母、数字、下划线和连字符。
- 记录文件名只能包含英文字母、数字、下划线和连字符，可带 `.json`。
- 插图参数只填写文件名；渲染器会补全固定目录 `data/attachments/`。文件名不能包含路径分隔符，并须使用 `png`、`jpg`、`jpeg`、`gif`、`webp` 或 `svg` 扩展名。
- 兼容层仍接受旧的 `data/attachments/文件名` 写法，但新内容不要再写完整路径。
- 未知类型、缺少参数或未闭合的标记会安全地作为普通文本显示。

## 迁移和检查

```powershell
node scripts/migrate-record-markup.mjs
node scripts/migrate-record-markup.mjs --check
node scripts/test-record-markup.mjs
```

迁移器遍历 `data` 下的全部 JSON 字符串字段。`--check` 只检查，不写文件。
